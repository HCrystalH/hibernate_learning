# Movie Theater Database Schema

## Table: CINEMA_ROOM

Stores information about cinema rooms.

| Column Name       | Data Type     | Constraints |
|------------------|--------------|-------------|
| CINEMA_ROOM_ID   | int          | Primary Key |
| CINEMA_ROOM_NAME | varchar(255) | Unique, Not Null |
| SEAT_QUANTITY    | int          | Not Null |

---

## Table: SEAT

Stores seat information for each cinema room.

| Column Name    | Data Type     | Constraints |
|---------------|--------------|-------------|
| SEAT_ID       | int          | Primary Key |
| CINEMA_ROOM_ID| int          | Foreign Key → CINEMA_ROOM.CINEMA_ROOM_ID |
| SEAT_COLUMN   | varchar(255) | Not Null |
| SEAT_ROW      | int          | Not Null |
| SEAT_STATUS   | varchar(255) | Not Null |
| SEAT_TYPE     | varchar(255) | Not Null |

### Relationship
- One **CINEMA_ROOM** can have many **SEAT** records.
- `SEAT.CINEMA_ROOM_ID` references `CINEMA_ROOM.CINEMA_ROOM_ID`.

---

## Table: CINEMA_ROOM_DETAIL

Stores room pricing and descriptive information over time.

| Column Name           | Data Type     | Constraints |
|----------------------|--------------|-------------|
| CINEMA_ROOM_DETAIL_ID| int          | Primary Key |
| CINEMA_ROOM_ID       | int          | Foreign Key → CINEMA_ROOM.CINEMA_ROOM_ID |
| ROOM_RATE            | int          | Not Null |
| ACTIVE_DATE          | date         | Not Null |
| ROOM_DESCRIPTION     | varchar(250) | Not Null |

### Relationship
- One **CINEMA_ROOM** have one**CINEMA_ROOM_DETAIL** record.
- `CINEMA_ROOM_DETAIL.CINEMA_ROOM_ID` references `CINEMA_ROOM.CINEMA_ROOM_ID`.

---
